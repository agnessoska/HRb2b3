# План улучшения дизайна страницы авторизации

## 1. Анализ текущего состояния
- Страница авторизации использует стандартные компоненты `Tabs` из `shadcn/ui`.
- Вкладки "Вход" и "Регистрация" выглядят стандартно и могут не сочетаться с общим стилем приложения.
- Формы входа и регистрации находятся внутри `Card`, что создает лишние рамки и отступы.
- Отсутствует визуальная иерархия и акценты.

## 2. Предлагаемые улучшения
- **Упрощение структуры:** Убрать `Card` внутри `TabsContent` для более чистого вида.
- **Стилизация вкладок:** Сделать переключатель вкладок более современным (например, `segment control` стиль).
- **Улучшение типографики:** Использовать более выразительные заголовки и описания.
- **Анимации:** Добавить плавные переходы между вкладками с использованием `framer-motion`.
- **Единообразие полей ввода:** Сделать поля ввода более "воздушными" и удобными.
- **Улучшение кнопок:** Сделать кнопки более заметными и интерактивными.
- **Социальные кнопки:** Выделить кнопку входа через Google.

## 3. План реализации
1.  Создать новый компонент `AuthTabs` для стилизованного переключения между режимами.
2.  Обновить `AuthForm.tsx` для использования нового компонента `AuthTabs`.
3.  Переработать верстку форм входа и регистрации, убрав лишние вложенности.
4.  Добавить анимации появления форм.
5.  Проверить адаптивность на мобильных устройствах.

## 4. Ожидаемый результат
- Более современный и привлекательный дизайн страницы авторизации.
- Улучшенный UX за счет плавных переходов и понятной навигации.
- Визуальное соответствие общему стилю приложения ("Zoomer-friendly").
